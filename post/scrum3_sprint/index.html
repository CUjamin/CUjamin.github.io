冲刺 Sprint

冲刺：Scrum在最长一个月的迭代或者周期中安排工作，这些迭代或者周期称为“冲刺”。

关键特性：

  • 冲刺是在时间盒内完成的。
  • 持续时间 短并且长度一致。
  • 冲刺开始后就不能再改变目标。
  • 必须达到团队的完成定义中要求的最终状态。

概述

时间限定

时间盒：每个冲刺都发生在一定的时间期限之内，有明确的开始日期和结束日期，称为一个时间盒。

  1. 设定WIP数量限制

    WIP：表示已经开始但尚未结束的工作清单。

  2. 强制排列优先级顺序

    可以将注意力集中在快速完成有价值的事情。

  3. 展示进度

  4. 避免不必要的完美主义

  5. 促进结束

  6. 增强可预测性

持续期短

  1. 容易规划

  2. 反馈快

  3. 投入产出比高

  4. 错误有限

  5. 有助于“满血复活”

  6. 检查点多

一致的持续期

  1. 有节奏感

  2. 简化规划活动

锁定冲刺目标

  1. 冲刺目标

  2. 共同承诺

  3. 是变更，还是澄清？

  4. 变更引起的后果

  5. 注重失效

  6. 异常终止

完成的定义

  1. 什么是完成的定义？

  2. 完成的定义可以随时间演变

  3. 完成的定时还是接受标准？

  4. 完成还是完成 - 完成
